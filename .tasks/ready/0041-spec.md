# 0041 -- Spec: `great mcp test <name>` shows wrong error when no `[mcp]` section exists

- **Task:** 0041
- **Type:** bugfix
- **Priority:** P3
- **Complexity:** XS (single conditional change, ~5 lines of production code)
- **Author:** Lovelace (Spec Writer)
- **Date:** 2026-03-04

## 1. Problem Statement

When a user runs `great mcp test myserver` against a `great.toml` that has no
`[mcp]` section (or an empty one), the CLI prints:

```
! No MCP servers declared in great.toml.
```

This is misleading because the user explicitly named a server. The correct
message -- already implemented at line 192 of `src/cli/mcp.rs` -- is:

```
x MCP server 'myserver' not found in great.toml
```

That message is unreachable when the MCP map is empty because the early-exit
guard on line 183 returns before the per-name lookup on line 189.

## 2. Root Cause Analysis

File: `/home/isaac/src/sh.great/src/cli/mcp.rs`, function `run_test()` (line 167).

```rust
// Line 181: unwrap the Option<HashMap> to an empty HashMap
let mcps = cfg.mcp.unwrap_or_default();

// Line 183-186: unconditional early exit when map is empty -- BUG
if mcps.is_empty() {
    output::warning("No MCP servers declared in great.toml.");
    return Ok(());
}

// Line 188-197: per-name lookup -- UNREACHABLE when mcps is empty
let servers_to_test: Vec<(&String, &crate::config::schema::McpConfig)> = match name {
    Some(n) => match mcps.get_key_value(n) {
        Some(pair) => vec![pair],
        None => {
            output::error(&format!("MCP server '{}' not found in great.toml", n));
            return Ok(());
        }
    },
    None => mcps.iter().collect(),
};
```

The guard at line 183 does not consider whether `name` was provided. When
`name` is `Some("myserver")` and `mcps` is empty, the guard fires and emits
the generic warning instead of allowing the code to fall through to the
name-specific error at line 192.

## 3. Exact Code Change

### File: `/home/isaac/src/sh.great/src/cli/mcp.rs`

**Before** (lines 183-186):
```rust
    if mcps.is_empty() {
        output::warning("No MCP servers declared in great.toml.");
        return Ok(());
    }
```

**After:**
```rust
    if mcps.is_empty() && name.is_none() {
        output::warning("No MCP servers declared in great.toml.");
        return Ok(());
    }
```

**Rationale:** Adding `&& name.is_none()` means the early-exit only fires when
no specific server was requested. When a name is provided, execution continues
to the `match name` block at line 188, where `mcps.get_key_value(n)` returns
`None` (because the map is empty), and the existing error message at line 192
fires correctly.

### No other files change.

### Function signature (unchanged):
```rust
fn run_test(name: Option<&str>) -> Result<()>
```

### Output function signatures (unchanged, for reference):
```rust
// src/cli/output.rs
pub fn warning(msg: &str)    // writes to stderr with yellow "!" prefix
pub fn error(msg: &str)      // writes to stderr with red "x" prefix
```

## 4. Behavioral Matrix

| Scenario | `name` | `mcps` | Before (broken) | After (fixed) |
|---|---|---|---|---|
| Named, no `[mcp]` section | `Some("xyz")` | empty | "No MCP servers declared" (wrong) | "MCP server 'xyz' not found in great.toml" (correct) |
| Named, servers exist, not this one | `Some("xyz")` | non-empty | "MCP server 'xyz' not found" | same (no change) |
| No name, no `[mcp]` section | `None` | empty | "No MCP servers declared" | same (no change) |
| No name, servers exist | `None` | non-empty | tests all servers | same (no change) |
| Named, server exists | `Some("fs")` | has "fs" | tests "fs" | same (no change) |

## 5. Edge Cases

1. **No `great.toml` at all:** The `config::discover_config()` call at line 171
   fails, hitting the `Err(_)` branch at line 175 which prints
   `"No valid great.toml found."` and returns `Ok(())`. This path is unaffected
   by the change.

2. **`great.toml` exists but is invalid TOML:** Same path as above -- the
   `config::load()` call returns `Err`, caught by `and_then` at line 172.

3. **`[mcp]` section exists but is empty table (`[mcp]` with no entries):**
   `cfg.mcp` is `Some(HashMap::new())`, so `unwrap_or_default()` gives an
   empty map. With the fix, `name.is_some()` will skip the guard, and
   `get_key_value` returns `None`, producing the name-specific error. Correct.

4. **Server name is an empty string:** `great mcp test ""` -- clap will pass
   `Some("")`. The `get_key_value("")` lookup returns `None`, producing
   `"MCP server '' not found in great.toml"`. Acceptable -- empty names are
   not a valid use case and the message is still actionable.

5. **Platform differences (macOS ARM64/x86_64, Ubuntu, WSL2):** This change
   is pure control flow on in-memory data structures. No platform-specific
   behavior involved.

## 6. Error Handling

All error paths in `run_test()` use `output::error()` or `output::warning()`
and return `Ok(())` (exit code 0). This spec does not change exit codes --
that is tracked separately in task 0040.

## 7. Security Considerations

None. This change modifies only the conditional that selects which diagnostic
message to print. No user input is passed to commands, file paths, or network
calls.

### 8.1 Integration Tests (add to `tests/cli_smoke.rs`)

Four new tests, one for each row in the acceptance criteria.

```rust
// -----------------------------------------------------------------------
// MCP Test — error message regression tests (task 0041)
// -----------------------------------------------------------------------

/// AC1: Named server, no [mcp] section -> name-specific error
#[test]
fn mcp_test_named_no_mcp_section_shows_name_error() {
    let dir = TempDir::new().unwrap();
    std::fs::write(
        dir.path().join("great.toml"),
        r#"
[project]
name = "test"
"#,
    )
    .unwrap();

    great()
        .current_dir(dir.path())
        .args(["mcp", "test", "nonexistent_xyz"])

}

[project]
name = "test"

[mcp.filesystem]
command = "echo"
"#,

    .unwrap();

    great()
        .current_dir(dir.path())

        .assert()
        .success()
```
