# Spec 0038 -- Handle SIGPIPE/EPIPE Gracefully

**Author:** Lovelace | **Date:** 2026-03-04 | **Approach:** Option A (Nightingale-recommended)

## Problem

Rust does not install a default SIGPIPE handler. When a downstream pipe reader closes early (e.g. `great status --json | head -0`), `println!` returns `io::Error(BrokenPipe)`, which propagates through `anyhow::Result` to an exit-101 panic with a confusing stderr message. Every subcommand that writes to stdout is affected.

## Solution

Add one attribute to `fn main()` in `src/main.rs` (line 13):

```rust
#[cfg_attr(unix, unix_sigpipe = "sig_dfl")]
fn main() -> Result<()> {
```

This restores the POSIX default SIGPIPE disposition. The kernel terminates the process with SIGPIPE (exit 141) before the `io::Error` is ever created. The `cfg_attr(unix, ...)` guard makes this a no-op on Windows.

**Stability:** `unix_sigpipe` was stabilized in Rust 1.75.0 (2023-12-28). This project has no explicit `rust-version` in `Cargo.toml` and depends on crates (rmcp 0.16, schemars 1.0) that require toolchains well past 1.75. No compatibility concern.

**File changed:** `src/main.rs` -- one line added before `fn main()`.

No other files change. No new dependencies.

## Test Plan

Manual verification (these cannot be automated in `cargo test` because they require a real pipe and signal delivery):

```bash
great status --json | head -0        # expect exit 0 or 141, stderr empty
great status --json | head -1        # expect exit 0 or 141, stderr empty
great doctor | head -0               # expect exit 0 or 141, stderr empty
great diff | head -0                 # expect exit 0 or 141, stderr empty
great status --json                  # expect exit 0, valid JSON (no regression)
cargo test                           # expect all tests pass
```

## Acceptance Criteria

1. `great status --json | head -0` exits 0 or 141 with no stderr output.
2. `great status --json | head -1` exits 0 or 141 with no stderr output.
3. At least two other subcommands (`great doctor`, `great diff`) do not panic on `| head -0`.
4. Normal operation unaffected: `great status --json` produces valid JSON and exits 0.
5. `cargo test` passes with no regressions.

## Out of Scope

- MCP bridge SIGPIPE handling (long-lived server process; different semantics).
- Windows behavior (no SIGPIPE on Windows; attribute is no-op via `cfg_attr`).
- Exit code policy changes for non-EPIPE error paths.
